算法学习 优先队列

算法讲解_极客时间 优先队列

时间：20190513

leetcode: 703、239

优先队列

堆栈：FILO

队列：FIFO

PriorityQueue:正常入，按照优先级出

实现机制：heap,binary search tree。（堆，二叉搜索树）

题目1 判断实时第k大的元素 leetcode#703

  • 题目
  • 思路

1）保存前k个最大值，排序，淘汰最小的，时间复杂度O(N* Klog(K))。

2)优先队列：利用小顶堆，保持元素个数为K个。新元素加入时比较堆的元素个数和k ，保证堆中元素个数不超过k，然后比较新加入元素与堆顶元素大小，大于堆顶元素则加入，小于则跳过。时间复杂度：N（1 or log2(K)）, O(Nlog2(K))。单次操作加速。

说明：python中堆的实现使用heapq实现小顶堆

  • 实例

  • 执行结果

执行用时 : 100 ms, 在Kth Largest Element in a Stream的Python3提交中击败了75.69% 的用户

内存消耗 : 17 MB, 在Kth Largest Element in a Stream的Python3提交中击败了7.75% 的用户

题目2 滑动窗口最大值 leetcode#239

  • 题目
  • 思路

    1）使用优先队列，大顶堆，维护堆（删除离开的元素，加入新的元素），堆顶元素是结果，时间复杂度O(N*logK)
    2)使用双端队列deque，入队列，若前面的值小于后来的，清除前面的，大于则存下来。

  • 实例

  • 执行结果

执行用时 : 152 ms, 在Sliding Window Maximum的Python3提交中击败了79.30% 的用户

内存消耗 : 19.8 MB, 在Sliding Window Maximum的Python3提交中击败了77.44% 的用户

20190509 博客通过批处理文件更新尝试

1 修改github用户名，调整博客地址

之前的用户名太难记，访问博客比较麻烦，修改了一个输入熟练的用户名，之后修改博客对应仓库的仓库名为新用户名.github.io即可。

2 通过批处理文件更新

每次新建博客及更新均靠打开git bash命令行来进行，虽然常用命令较多，但还是比较麻烦，所以尝试写一个批处理文件，写完文件复制到_post文件夹下
，运行批处理文件完成自动更新。

2.1 第一次尝试

批处理文件内容

功能:打开git bash 并定位到需要的位置
尝试结果：能完成功能

2.2 第二次尝试

批处理文件内容

功能:不使用 git bash 尝试直接用命令行完成博客更新。

尝试结果：不能更新

2.3 第三次尝试

批处理文件内容

功能:不使用 git bash 尝试直接用命令行完成博客更新。

尝试结果：失败

查找问题：修改用户名后为了域名的简洁修改了.github.io 的仓库名，但是在_config.yml未修改仓库名，不能同步上去。

解决：修改_config.yml中地址为当前地址

结果：成功

2.4 第四次尝试

批处理文件内容

功能:不打开 git bash 尝试直接用命令行完成博客更新。

尝试结果：成功 撒花庆祝

3 思考

其实第二次尝试应该就能够成功的，不过自己的配置出了点问题，且当时没有详细看执行结果中的错误信息，导致花了更多的时间在网络上进行查找，最终还是自己解决了问题。

官方文档和执行报错真的是应该花时间研究的，不能一遇到问题不思考就直接百度/google答案。
